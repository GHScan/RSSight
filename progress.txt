# Progress

## Log (iteration history)

Do not keep a per-iteration log here. Use `git log` (e.g. by story id, tag, or commit message) to see what was done in each iteration.

## Lessons learned

Items below are lessons that affect subsequent development. Append here only when an iteration yields something worth recording for future work.

- [meta] Only after all tests and local quality checks pass may cross-story state (progress.txt, prd.json `passes`) be updated. Do not mark a story complete or append here while tests are failing or not run.
- [bootstrap] Business features start from the first story with `passes=false` in prd.json; earlier work was repo and infrastructure only.
- [frontend] Testing pages that fetch on mount: in jsdom there is no real backend, so unmocked fetch fails and the page stays in loading or error. Mock the API layer (e.g. vi.mock("../api/client")) and use waitFor for async state, or route/content assertions will flake or fail. Later work on new pages or tests may hit this again if mocks are omitted.
- [frontend] Routing tests that render the app at different routes in the same file can leave multiple roots in the document; the next test then sees "Found multiple elements" for the same heading/link. Call cleanup() in beforeEach in that test file so the DOM is cleared between tests.
- [shell / Windows] Do not use "&&" to chain commands: the project targets Windows, and in PowerShell (common in dev environments) "&&" is not valid and causes "The token '&&' is not a valid statement separator" (or similar). Root cause: assuming Unix/bash syntax. Use ";" to chain in PowerShell, or run one command per invocation, or use .cmd/.bat where "&" is used for chaining.
